# `prompt_builder.py`

## 檔案目的

`prompt_builder.py` 是一個專門用於**動態載入代理人角色卡（Role Cards）**的輔助模組。它的核心職責是根據傳入的代理人名稱（`agent_name`）和策略（`policy`），從 `prompt/` 資料夾中讀取對應的 Markdown (`.md`) 檔案，並將其內容作為系統提示（System Prompt）返回。

這個模組將提示管理的邏輯（如何根據策略尋找和讀取檔案）與代理人工作流的執行邏輯（如 `supervisor_workflow.py`）分開，提高了程式碼的模組化程度和可維護性。

## 主要功能

*   **`get_system_prompt(agent_name: str, policy: str)`**: 這是此模組提供的唯一函式。其工作流程如下：
    1.  **構建檔案路徑**: 根據 `agent_name` 和 `policy`，它會構建兩個可能的檔案名稱：
        *   帶有策略後綴的特定版本：`{agent_name}_card_{policy}.md` (例如, `supervisor_card_strict.md`)
        *   一個通用的基礎版本：`{agent_name}_card.md` (例如, `supervisor_card.md`)
    2.  **路徑解析**: 它會將上述檔案名稱與寫死在程式碼中的 `PROMPTS_DIR` (`prompt/` 資料夾) 結合，形成完整的文件路徑。
    3.  **優先級載入**: 函式會**優先**嘗試載入帶有策略後綴的特定版本角色卡。如果該檔案不存在，它會**回退**到載入通用的基礎版本角色卡。
    4.  **錯誤處理**: 如果兩個版本的角色卡都不存在，它會打印一條錯誤訊息，並返回一個非常基礎的、通用的預設提示，以確保系統不會因此崩潰。
    5.  **內容安全**: 在讀取檔案內容後，它會將 `{{` 和 `}}` 替換為 `{{{{` 和 `}}}}`，這是一種安全措施，以防止檔案內容中的大括號 `_` 與 Python 的字串格式化（f-string）語法發生衝突。

## 設計理念

*   **配置與程式碼分離**: 將作為「配置」的提示文本（Prompt Text）從作為「邏輯」的 Python 程式碼中分離出來，並放入專門的 `.md` 檔案中。這使得修改提示變得非常容易，無需觸碰任何 Python 程式碼，也方便非開發人員進行審閱和編輯。
*   **策略化加載**: 支援根據不同的 `policy` 載入不同的角色卡，這為進行 A/B 測試或在不同實驗條件下賦予代理不同行為模式提供了極大的靈活性。
*   **路徑穩健性**: 使用 `os.path.dirname(__file__)` 來定位 `prompt/` 資料夾，確保了無論從哪個目錄執行主程式，都能正確地找到提示檔案，避免了相對路徑帶來的問題。

## 依賴關係

*   **Python 函式庫**: `os`。

## 在專案中的角色

`prompt_builder.py` 是 MAS 系統中**代理人「性格」和「指令」的來源**。每個專家代理和 Supervisor 的行為模式、工作流程和限制，很大程度上是由它們的角色卡（System Prompt）定義的。這個模組確保了每個代理在被喚醒時，都能被賦予正確的、與當前實驗策略相符的「靈魂」。它是實現代理行為定製化和實驗變量控制的關鍵一環。
